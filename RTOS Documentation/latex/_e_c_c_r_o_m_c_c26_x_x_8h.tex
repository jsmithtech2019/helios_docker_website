The E\+C\+C\+R\+O\+M\+C\+C26\+XX driver provides reentrant access to the R\+OM based E\+CC module. Access is given first come, first serve and subsequent calls block until all preceding callers have returned or the caller times out in which case execution returns to the caller but the operation is not performed.\hypertarget{_e_c_c_r_o_m_c_c26_x_x_8h_autotoc_md10}{}\doxysubsubsection{General Behavior}\label{_e_c_c_r_o_m_c_c26_x_x_8h_autotoc_md10}

\begin{DoxyItemize}
\item Prior to calling either \mbox{\hyperlink{_e_c_c_r_o_m_c_c26_x_x_8h_a97cdc759b911e36c7ccc18cfc45fede5}{E\+C\+C\+R\+O\+M\+C\+C26\+X\+X\+\_\+gen\+Keys()}} or \mbox{\hyperlink{_e_c_c_r_o_m_c_c26_x_x_8h_a64032ff4d46e71fcc876eb0f0491f13c}{E\+C\+C\+R\+O\+M\+C\+C26\+X\+X\+\_\+gen\+D\+H\+Key()}}, parameters must be initialized with a call to \mbox{\hyperlink{_e_c_c_r_o_m_c_c26_x_x_8h_ab4b43f73cb42748bf9261a43953f8709}{E\+C\+C\+R\+O\+M\+C\+C26\+X\+X\+\_\+\+Params\+\_\+init()}} and initialize an instance of \mbox{\hyperlink{struct_e_c_c_r_o_m_c_c26_x_x___params}{E\+C\+C\+R\+O\+M\+C\+C26\+X\+X\+\_\+\+Params}}. Multiple clients may share the same \mbox{\hyperlink{struct_e_c_c_r_o_m_c_c26_x_x___params}{E\+C\+C\+R\+O\+M\+C\+C26\+X\+X\+\_\+\+Params}} instance as long as they do not rely on the status field. This field could be overwritten by any of the clients whom share the instance. The clients should then rely on the return value of the A\+PI function which was called.
\item \mbox{\hyperlink{struct_e_c_c_r_o_m_c_c26_x_x___params}{E\+C\+C\+R\+O\+M\+C\+C26\+X\+X\+\_\+\+Params}} initialize their curves to N\+I\+ST P-\/256 defaults, set their malloc and free functions to N\+U\+LL and block indefinitely when waiting for the resource.
\item Currently only N\+I\+ST P-\/256 parameters are included in R\+OM. If a client wishes to use a different curve, that client must provide it. Curve parameters are uint32\+\_\+t buffers where the first uint32\+\_\+t block is a length header which specified the length of the buffer in uint32\+\_\+t blocks, not including the header itself. the maximum curve length supported in 1020 bytes or 255 uint32\+\_\+t blocks. For example, the N\+I\+ST P-\/256 length header would be 8.
\item key\+Len should be the same as the length header of Curve parameters but in in bytes. For example, the N\+I\+ST P-\/256 key length would be 32.
\item workzone length is determined by the operation, curve length in uint32\+\_\+t blocks and windowsize. the default value for N\+I\+ST P-\/256 curves is only valid for those curves performing key and shared secret generation. The workzone length for any curve using the supported operation can be calculated with the E\+C\+C\+R\+O\+M\+C\+C26\+X\+X\+\_\+\+W\+O\+R\+K\+Z\+O\+N\+E\+\_\+\+L\+EN macro.
\item window\+Size determines the speed and memory trade-\/off of each operation. Valid window\+Size values are 2 through 5, and 3 is recommended as the best trade-\/off.
\end{DoxyItemize}
